17  /**
18  * @brief Class <code>SerializeIO</code> is a sentry class which performs serialization
19  * for mesh database I/O.
20  *
21  * This sentry guards serialization of parallel I/O routines. At construction, it
22  * blocks the processes via an MPI barrier, releasing them to execute in groups specified
23  * by <code>s_groupSize</code>. At destruction, it continues to block via MPI barriers
24  * until all the processor have been released by the constructor.
25  *
26  * In the case where the constructor is called, and the sentry is already active and owned
27  * by the processes group, the constructor and destructor simply fall through since the
28  * serialization is already in place at a higher level.
29  *
30  * \note All ranks must call the SerializeIO constructor synchronously.
31  * \note It is recommended to use RAII and keep the area protected by the SerializeIO as small as
32  * possible.
33  *
34  * The flow is that the ranks are split into groups of the specified size. Assume 3 ranks of group
35  * size 1.
36  *
37  * * First time through,
38  * - rank 0 falls through and
39  * - ranks 1, 2 sit at the barrier
40  * - rank 0 hits the destructor and then all 3 ranks are in the barrier so they all go to next
41  * step
42  * - (rank 1,2 in constructor, rank 0 in destructor)
43  * * `s_owner` is now equal to `m_groupRank` on rank 1, so it falls out of the do while;
44  * - rank 2 still in the constructor do while Barrier
45  * - rank 0 in the destructor do while Barrier
46  * - rank 1 does its work and calls destructor;
47  * - all ranks in Barrier, so they go to next step.
48  * * `s_owner` now equal to `m_groupRank` on rank 2, so if falls out of the do while;
49  * - ranks 0,1 in destructor do while at the Barrier
50  * - rank 2 does it work and calls destructor
51  * - all ranks are now in the destructor Barrier, so they go to next step
52  * - all ranks clear the Destructor and go to next step.
53  *
54  */
